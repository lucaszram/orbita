# Ritmo De Trabajo - Órbita

## Orden correcto

1. Leer `docs/contexto-actual.md`.
2. Leer `docs/figma-context.md` si la tarea toca diseno, Figma, assets o onboarding.
3. Cerrar copy exacto y estructura de pantalla.
4. Recién ahi editar Figma o codigo.
5. Verificar con capturas, busquedas o tests segun corresponda.

## Regla principal

No disenar antes de tener texto y funcion de pantalla definidos. En este proyecto, disenar primero suele terminar en pantallas lindas pero incorrectas.

## Pantalla por pantalla

- Cambiar una pantalla o bloque por vez.
- No rehacer todo el onboarding salvo pedido explicito.
- No mezclar dos funciones en una misma pantalla.
- No mover una referencia de pantalla a otra sin confirmar el orden del flujo.
- Si una pantalla ya fue corregida por el usuario, preservar esa decision.

## Copy

- Usar voseo: `vos`, `podes`, `miras`, `cancelas`.
- Preferir frases cortas y naturales.
- Evitar traducciones literales de Moonly/Co-Star.
- Evitar jerga rara como `mas tierra` si no suena natural.
- No usar ingles visible en UI final.

## Diseno

- Evitar cards, placeholders y grillas por defecto.
- Una pieza visual fuerte por pantalla suele ser mejor que muchas cajas.
- Usar aire, jerarquia, diagramas y UI editable.
- Mantener dark premium, charcoal, cobre y geometria orbital cuando aplique.
- No usar emojis coloridos nativos si rompen el tono editorial.

## Cierre de tarea

Antes de cerrar:

- Revisar que el copy aplicado sea el ultimo aprobado.
- Verificar que no haya textos viejos ocultos o visibles.
- Sacar screenshot si se toco Figma.
- Buscar terminos historicos si se edito documentacion.
- Explicar que cambio y que quedo fuera de alcance.
